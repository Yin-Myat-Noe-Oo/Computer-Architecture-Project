Algorithm Explanation for Project Presentation

Core Translation Algorithm

1

Input Processing

The algorithm begins by taking the input text and breaking it down into individual characters.

// Pseudo-code for input processing
inputText = "Hello"
characters = ['H', 'e', 'l', 'l', 'o']
charCodes = [72, 101, 108, 108, 111] // ASCII values
2

Memory Storage

Each character's ASCII value is stored in memory locations for processing.

// Memory allocation example
Memory[0x10] = 0x48 // 'H'
Memory[0x11] = 0x65 // 'e'
Memory[0x12] = 0x6C // 'l'
Memory[0x13] = 0x6C // 'l'
Memory[0x14] = 0x6F // 'o'
3

Instruction Execution

The CPU follows a program that loads, translates, and stores each character.

// Program instructions
0x00: LOAD 0x10 // Load 'H' from memory
0x01: TRANSLATE // Apply translation
0x02: STORE 0x18 // Store translated character
0x03: LOAD 0x11 // Load 'e' from memory
0x04: TRANSLATE // Apply translation
0x05: STORE 0x19 // Store translated character
// ... continues for each character

CPU Simulation Algorithm

1

Fetch Stage

The CPU retrieves the next instruction from memory.

PC (Program Counter): 0x00
0x00
MAR (Memory Address Register): 0x00
MDR (Memory Data Register): 0x01
IR (Instruction Register): 0x01
2

Decode Stage

The Control Unit interprets the instruction and activates appropriate signals.

READ
WRITE
FETCH
EXECUTE
3

Execute Stage

The CPU performs the operation specified by the instruction.

// Execution example for LOAD instruction
1. Read memory at address specified in next byte
2. Move data to Accumulator (AC) register
3. Increment Program Counter (PC) by 2

Control Unit Algorithm

1

Instruction Cycle

The Control Unit manages the fetch-decode-execute cycle:

while (running) { 1. FETCH: Get next instruction from memory 2. DECODE: Determine what instruction does 3. EXECUTE: Perform the operation 4. Update PC to next instruction }
2

Signal Management

The Control Unit activates specific signals for each operation:

if (opcode == LOAD) { activate(READ_SIGNAL); activate(MEM_TO_AC_SIGNAL); } else if (opcode == STORE) { activate(WRITE_SIGNAL); activate(AC_TO_MEM_SIGNAL); } else if (opcode == TRANSLATE) { activate(ALU_OP_SIGNAL); }

Translation Process

1

Character Processing

Each character goes through the translation pipeline:

Input: "Hello" (5 characters) Step 1: Fetch 'H' from memory Step 2: Apply translation algorithm Step 3: Store translated 'H' Repeat for each character
2

Simulation vs Real Translation

The simulator shows the process, then uses Google Translate API for actual translation:

// Simulation (for visualization) function simulateTranslation(charCode) { return charCode + 1; // Simple transformation } // Real translation (after simulation) async function realTranslation(text, sourceLang, targetLang) { const response = await fetchGoogleTranslateAPI(text, sourceLang, targetLang); return response.translatedText; }
